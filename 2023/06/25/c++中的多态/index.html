c++中的多态

多态

在C++中，多态是一种面向对象编程的概念，它允许基于对象的实际类型来调用相应的方法或函数。多态性允许您在不考虑对象类型的情况下编写通用的代码，以增加代码的灵活性和可重用性。

实现多态性的主要机制是通过虚函数（virtual functions）和基类指针或引用来实现。以下是使用多态性的基本步骤：

创建一个基类（base class），其中包含一个或多个虚函数。虚函数是在基类中声明为虚拟的函数，它们可以在派1. 生类（derived class）中进行重写（override）。

  1. 派生一个或多个类（derived class）从基类，并重写基类中的虚函数。在派生类中，使用相同的函数签名（函数名和参数列表）来重写基类中的虚函数。
  1. 使用基类指针或引用来引用派生类的对象。通过基类指针或引用，可以实现对派生类对象的多态访问。

在上述示例中，基类Shape有一个虚函数draw()，而派生类Circle和Rectangle重写了这个函数。通过使用基类指针shapePtr，我们可以在运行时决定调用哪个派生类的draw()函数，从而实现多态性。

需要注意的是，为了正确使用多态性，必须将基类的虚函数声明为virtual，并将派生类的函数重写声明为override，以确保函数的动态绑定。此外，通常建议将基类的析构函数声明为虚函数，以便在删除基类指针时正确调用派生类的析构函数。

静态绑定和动态绑定

静态绑定（地址早绑定）和动态绑定（地址晚绑定）是与多态性密切相关的概念，它们描述了在编译时和运行时确定函数调用的方式。

  1. 静态绑定（地址早绑定）：
    • 在地址早绑定中，函数调用在编译时就确定了，根据引用或指针的类型来确定调用哪个函数。
    • 在静态绑定中，编译器根据变量的声明类型来解析函数调用，它不考虑变量实际引用的对象的类型。
    • 静态绑定适用于非虚函数和静态函数。
  2. 动态绑定（地址晚绑定）：
    • 在晚绑定中，函数调用是在运行时基于对象的实际类型来确定的。
    • 动态绑定适用于虚函数，通过使用虚函数和基类指针或引用，可以实现在运行时根据对象的实际类型调用相应的函数。
    • 在动态绑定中，编译器在编译时不能确定函数调用的具体目标，因此在运行时会进行动态查找。

多态性可以通过动态绑定（晚绑定）来实现，而晚绑定依赖于虚函数和基类指针或引用。通过多态性，可以实现代码的灵活性和可重用性，以及在运行时根据对象的实际类型调用适当的函数。

多态性与动态绑定密切相关。通过将虚函数声明为基类的一部分，并使用基类指针或引用引用派生类对象，可以实现在运行时调用适当的函数，实现多态性。编译器会使用动态绑定来确定函数的目标，从而实现多态性的效果。

总结起来，多态性是通过动态绑定来实现的，而动态绑定允许在运行时根据对象的实际类型确定函数调用。静态绑定是与多态性相对应的概念，它是在编译时根据变量的声明类型确定函数调用。

重写的要求

在C++中，重写（Override）基类的虚函数有以下要求：

  1. 基类函数必须是虚函数：只有在基类中将函数声明为虚函数，才能在派生类中进行重写。虚函数通过在基类函数声明前加上 virtual 关键字来声明。
  2. 函数签名必须相同：派生类中的重写函数必须与基类函数具有相同的函数签名（函数名和参数列表），包括返回类型。函数签名不同的函数将被视为派生类中的新函数，而不是重写基类函数。
  3. 访问权限不能更窄：派生类中的重写函数的访问权限不能比基类中的函数更窄。如果基类函数是 public，那么派生类中的重写函数可以是 public 或 protected，但不能是 private。
  4. 使用 override 关键字（可选）：在派生类中的重写函数声明中，可以使用 override 关键字来显式地指示该函数是对基类函数的重写。这是为了提高代码的可读性，并帮助检测错误，如果派生类中的函数签名与基类函数不匹配，编译器将会报错。

纯虚函数和抽象类

纯虚函数（Pure Virtual Function）

  • 纯虚函数是在基类中声明的虚函数，但没有提供具体的实现。
  • 通过在函数声明末尾使用 “= 0” 来将函数声明为纯虚函数。
  • 纯虚函数的存在意味着基类只定义了接口，而没有提供实际的函数实现。
  • 派生类必须重写（Override）纯虚函数，以提供自己的实现。
  • 带有纯虚函数的类被称为抽象类，它不能被实例化，只能被用作其他类的基类。

抽象类（Abstract Class）

  • 抽象类是包含至少一个纯虚函数的类。
  • 抽象类用于定义接口和规范派生类的行为。
  • 抽象类不能被实例化，只能被用作其他类的基类。
  • 派生类必须实现抽象类中的所有纯虚函数，才能被实例化，否则还是抽象类。

抽象类提供了一种将相关的类组织起来并定义共同接口的方式，它通过纯虚函数定义接口，但没有提供实际的函数实现。派生类必须重写纯虚函数来提供具体的实现，以使抽象类可以被实例化。

需要注意的是，如果派生类没有重写抽象类中的所有纯虚函数，那么该派生类也将被视为抽象类，不能被实例化。

总结：纯虚函数是在基类中声明但没有实现的虚函数，它用于定义接口并使类成为抽象类。抽象类是包含至少一个纯虚函数的类，它不能被实例化，只能用作其他类的基类。派生类必须重写抽象类中的纯虚函数才能被实例化。

纯虚析构函数

问题

当父类指针指向子类对象并使用多态时，如果父类中的析构函数不是虚析构函数或纯虚析构函数，那么在通过父类指针释放对象时，只会调用父类的析构函数，而不会调用子类的析构函数。这可能导致子类中在堆区（动态分配的内存）开辟的资源无法被正确释放，从而导致内存泄漏或其他问题。

简单的说就是， 使用多态时， 如果子类中有属性开辟到堆区。 那么父类指针在释放时无法调用到子类的析构代码，可能导致内存；那么解决方法就是，将父类中的析构函数改为虚析构或者纯虚构。
