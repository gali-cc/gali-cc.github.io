C++中的继承

继承

​ 继承是面向对象编程中的一种机制，它允许一个类（称为子类或派生类）从另一个类（称为父类或基类）继承属性和方法。

​ 通过继承，子类可以获得父类的属性和方法，并且可以在此基础上添加自己的特定属性和方法，从而实现代码的重用和扩展。继承是面向对象编程的核心概念之一，它支持代码的组织、抽象和封装。

​ 通过继承，子类可以继承父类的公有成员和保护成员，但无法直接访问父类的私有成员。子类可以在自己的定义中添加新的成员和方法，也可以重写（override）父类的方法以实现特定的行为。

​ 继承可以形成类的层次结构，其中一个类可以有多个子类，子类又可以有自己的子类，形成一个继承链。这种继承链可以提供代码的模块化、封装和多态等特性，使得代码更易于理解、维护和扩展。

​ 总结起来，继承是一种通过子类从父类继承属性和方法的机制，它是面向对象编程的重要概念之一，用于实现代码的重用、扩展和组织。

继承方式

在C++中，有三种主要的继承方式：

公有继承

公有继承（Public Inheritance）： 公有继承是最常见的继承方式。在公有继承中，基类的公有成员在派生类中仍然是公有的，基类的保护成员在派生类中变为保护的，私有成员在派生类中无法直接访问。公有继承使用关键字”public”。如果没有显式指定继承方式，默认为公共继承。

保护继承

保护继承（Protected Inheritance）： 保护继承与公有继承相似，但基类的公有和保护成员在派生类中都变为保护的。私有成员在派生类中无法直接访问。保护继承使用关键字”protected”。

私有继承

私有继承（Private Inheritance）： 私有继承中，基类的公有和保护成员在派生类中都变为私有的。私有继承主要用于实现”实现继承”或”继承接口”的概念，而不是用于基类的功能扩展。私有继承使用关键字”private”。

​ 在三种常见的继承方式（公有继承、私有继承和保护继承）中， 父类中私有成员属性有被继承，但编译器把它隐藏了， 所以子类都不能直接访问父类的私有成员。

​ 这些继承方式决定了派生类中基类成员的可见性和访问权限。需要注意的是，无论继承方式如何，派生类都可以通过相应的访问控制符（public、protected、private）重新调整成员的可见性。

构造与析构

在C++中，当存在继承关系时，对象的构造和析构顺序遵循以下规则：

  1. 构造顺序：
    • 首先，调用基类的构造函数。构造函数按照继承的顺序依次调用，从最顶层的基类开始，一直到最底层的派生类。
    • 在每个类的构造函数中，先执行该类自身的构造代码，然后再调用其成员对象的构造函数，最后执行构造函数的函数体。
    • 如果类的构造函数没有显式调用基类的构造函数，编译器会自动调用基类的默认构造函数。
  2. 析构顺序：
    • 首先，调用派生类的析构函数。析构函数按照继承的顺序相反的顺序依次调用，从最底层的派生类开始，一直到最顶层的基类。
    • 在每个类的析构函数中，先执行析构函数的函数体，然后再调用其成员对象的析构函数，最后执行该类自身的析构代码。
    • 如果类的析构函数没有显式调用基类的析构函数，编译器会自动调用基类的析构函数。

同名成员

在C++中，如果派生类继承了一个基类，并且派生类中定义了与基类中同名的成员，那么派生类的同名成员将隐藏基类的同名成员。这种情况被称为名称隐藏（name hiding），这适用于静态成员和非静态成员。

具体来说，当派生类中定义了与基类中同名的函数时，基类的同名函数将被隐藏，并且在派生类的作用域中无法直接访问。这意味着在派生类的对象上调用同名函数时，会优先调用派生类中的函数，而不会调用基类中的函数。

但是，如果希望在派生类中调用基类的同名函数，可以使用作用域解析运算符::来指定基类作用域，并显式调用基类的函数。例如：

  • 非静态成员的隐藏： 当派生类中定义了与基类中同名的非静态成员（变量或函数），基类的同名成员会被隐藏。通过派生类的对象无法直接访问基类中被隐藏的同名成员，除非通过类型转换或作用域解析运算符::来显式访问。
  *如果派生类中出现和基类同名的成员函数， 派生类的同名成员会隐藏掉基类中**所有**同名函数，也就是包括基类同名的重载的函数*
  • 静态成员的隐藏： 当派生类中定义了与基类中同名的静态成员（变量或函数），基类的同名静态成员会被隐藏。通过派生类或基类的作用域无法直接访问被隐藏的同名静态成员，除非通过类型转换或作用域解析运算符::来显式访问。

总结起来，无论是静态成员还是非静态成员，当派生类中定义了与基类中同名的成员时，基类的同名成员都会被隐藏。要访问被隐藏的同名成员，可以使用类型转换或作用域解析运算符::来显式指定基类的作用域。

虚继承（补充）

虚继承的底层实现涉及虚基类表（Virtual Base Class Table）和虚基类指针（Virtual Base Class Pointer）。

当一个类使用虚继承时，派生类中会包含一个指向虚基类表的指针。虚基类表是一个数据结构，用于存储虚基类相关的信息，包括虚基类的偏移量和访问权限等。

每个虚基类只在派生类的最终派生类（也称为“最底层派生类”）中有唯一的实例，而不会在继承链中的每个派生类中都有实例。这样就避免了多个实例的数据冗余和访问二义性。

在虚继承的情况下，派生类对象中的虚基类指针指向该对象中共享的虚基类实例。通过虚基类指针，可以在派生类中访问虚基类的成员。

在虚继承中，派生类构造函数负责初始化虚基类，而不是基类构造函数。这是因为虚基类只有一个实例，需要确保它只被初始化一次。派生类构造函数在初始化列表中指定虚基类的构造函数，并负责将虚基类指针指向正确的位置。

在上面的例子中，Platypus类使用虚继承从Mammal和Bird两个基类继承，而这两个基类又共同继承自虚基类Animal。因此，Platypus类中将包含一个指向虚基类表的指针，用于处理虚基类的访问和偏移。

虚基类表是编译器在编译阶段生成的数据结构，它记录了虚基类在派生类中的偏移量以及其他相关信息。这个表位于派生类对象的内存布局中的固定位置。

关于Platypus类的大小为什么是20字节（bytes）

首先，Platypus类继承了两个基类：Mammal和Bird。这两个基类都使用了虚继承，因此在派生类Platypus中，它们的子对象中都包含了一个指向虚基类Animal的指针。由于每个指针需要占用8字节（64位系统），所以这两个基类子对象的总大小为16字节（8字节的指针 + 4字节的对齐成员 + 4字节的填充）。

然后，Platypus类还有自己的成员变量和可能的填充字节。在这个例子中，只有一个int类型的成员变量age，它占用4字节。

因此，Platypus类的总大小为16字节（基类子对象的大小） + 4字节（成员变量age的大小） = 20字节。

需要注意的是，类的大小可能受到编译器和平台的影响，对齐规则和填充字节的添加可能会导致实际的类大小与所预期的不同。

当派生类对象需要访问虚基类的成员时，编译器使用虚基类指针和虚基类表来确定正确的偏移量，并通过偏移量访问虚基类的成员。

通过虚继承和虚基类表/指针的组合，C++实现了在多继承中共享虚基类的实例，避免了数据冗余和访问二义性的问题。这种机制确保了在派生类对象中正确访问虚基类的成员，并提供了了一致和可靠的行为。
