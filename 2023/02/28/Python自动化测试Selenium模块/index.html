简介

Python的Selenium库是一个用于Web应用程序自动化测试的工具。它可以通过模拟用户在浏览器中的操作来测试Web应用程序的功能，比如点击按钮、填写表单、导航到不同的页面等。它支持多种浏览器，包括Chrome、Firefox、Edge等。

1. 定位元素

Selenium提供了多种方式来定位页面元素，包括：

  • 根据元素ID查找：driver.find_element(by="id", value="element_id")
  • 根据元素名称查找：driver.find_element(by="name", value="element_name")
  • 根据元素链接文本查找：driver.find_element(by="link text", value="link_text")
  • 根据元素CSS选择器查找：driver.find_element(by="css selector", value="css_selector")
  • 根据元素XPath查找：driver.find_element(by="xpath", value="xpath_expression")

其中，CSS选择器和XPath是两种比较常用的定位方式，可以实现比较精确的定位。

也可以直接使用导入By来使用from selenium.webdriver.common.by import By,如driver.find_element(by=By.ID, value="element_id")

常用用法

1. 等待页面元素加载

在使用Selenium自动化测试时，我们经常会遇到需要等待页面元素加载完成后才能进行下一步操作的情况。为了解决这个问题，Selenium提供了一些等待页面元素加载的方法，例如：

  • driver.implicitly_wait(timeout)：隐式等待，在指定的时间内等待页面元素加载完成。
  • WebDriverWait(driver, timeout).until(EC.presence_of_element_located((By.ID, 'element_id')))：显式等待，等待指定的页面元素出现。

其中，EC是Selenium提供的一些预期条件，例如：presence_of_element_located表示等待元素出现，visibility_of_element_located表示等待元素可见等等。

以下是 “Expected Conditions” 模块中一些常用的条件：

  • presence_of_element_located：等待页面上的单个元素出现，并且至少存在于DOM中。
  • presence_of_all_elements_located：等待所有匹配给定定位器的元素出现在页面上，并且至少存在于DOM中。
  • visibility_of_element_located：等待单个元素在页面上可见。
  • visibility_of_all_elements_located：等待所有匹配给定定位器的元素在页面上可见。
  • element_to_be_clickable：等待元素可见并且可点击。
  • text_to_be_present_in_element：等待给定元素的文本出现。
  • title_is：等待页面标题与给定的文本匹配。
  • alert_is_present：等待页面上出现警报。

2. 模拟用户操作

Selenium可以模拟用户对页面的各种操作，例如：点击、输入、下拉选择等等。例如：

  • element.click()：模拟点击元素。
  • element.send_keys('text')：模拟在输入框中输入文本。
  • Select(element).select_by_index(index)：模拟下拉选择框中选择第几个选项。

3. 切换窗口和框架

在页面中，可能会存在多个窗口或者嵌套的框架，Selenium可以切换不同的窗口和框架，例如：

  • driver.switch_to.window(window_handle)：切换到指定的窗口。
  • driver.switch_to.frame(frame_reference)：切换到指定的框架。

切换的窗口句柄可以通过driver.window_handles获得。

4. 处理弹窗

有些页面会弹出弹窗，需要在弹窗中进行操作，Selenium可以处理这些弹窗，例如：

  • alert = driver.switch_to.alert：获取弹窗对象。
  • alert.accept()：点击弹窗中的确认按钮。
  • alert.dismiss()：点击弹窗中的取消按钮。

5. 执行JavaScript代码

有些页面的元素可能无法直接操作，需要使用JavaScript代码来完成，Selenium可以执行JavaScript代码，例如：

  • driver.execute_script('document.getElementById("element_id").value = "text"')：使用JavaScript代码来设置元素的值。
  • driver.execute_script('window.scrollTo(0, document.body.scrollHeight)')：使用JavaScript代码来滚动页面。
  • driver.execute_script("arguments[0].click();", element) 当Selenium元素定位成功，但是点击（click）操作无效时，可以尝试使用JavaScript代码来执行。

这些只是Selenium的一些常用用法，还有很多其他的用法，例如：处理Cookie、截图、添加代理等等。如果需要深入了解Selenium的使用，建议查看官方文档：https://www.selenium.dev/documentation/en/。

7. 使用其他第三方库

除了上述介绍的库之外，还有许多其他的第三方库可以用于Selenium测试。例如：

  • pyvirtualdisplay：一个虚拟显示器库，可以在没有图形界面的机器上运行浏览器。

  • requests：一个HTTP客户端库，可以用于发送HTTP请求。
  • pandas：一个数据处理库，可以方便地处理测试数据。

使用这些第三方库可以扩展Selenium测试的功能，并且可以更加方便地处理测试数据和结果。
